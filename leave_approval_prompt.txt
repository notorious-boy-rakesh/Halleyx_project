You are an advanced agentic coding assistant. I want you to build a full-stack **Leave Approval Automation System** using a sophisticated workflow execution engine. 

The project should be designed with a highly polished, corporate, enterprise-grade UI theme (clean white backgrounds, subtle soft shadows, rounded corners, professional blues/greys, modern typography). Provide a seamless professional experience.

### 1. Technology Stack
* **Backend:** Node.js, Express.js
* **Database:** MongoDB (Mongoose). Please handle local testing seamlessly by using `mongodb-memory-server` integrated via a `run_demo.js` bootstrapper.
* **Frontend:** Vanilla HTML, CSS, JS with Bootstrap 5 and Chart.js (Strictly no heavy frameworks like React/Next.js).

### 2. Core Architecture & Features
* **Role-Based Access Control (RBAC):** Admin (HR Lead), Manager (Approver), User (Employee). 
* **Dynamic Workflow Builder:** A custom visual, canvas-based drag-and-drop UI using vanilla JS and SVG connector lines. This allows HR to visually map out routing for leave requests.
* **Workflow Step Types:** 
   - **Task:** Automated action (e.g., Deduct Leave Balance, Update Payroll).
   - **Approval:** Human decision (e.g., Manager Review).
   - **Notification:** Messaging (e.g., Email employee with decision).
* **Safe Rule Engine:** A secure, custom expression parser (STRICTLY NO `eval()`) to evaluate conditions like `leaveDays > 5 && type == "Sick"`. Include an "AI Suggestion" endpoint to convert natural language to rule syntax.
* **Execution Engine:** A resilient, step-by-step runner that processes a submitted Leave Request (JSON payload) against the defined workflow structure, evaluating rules to determine the next step.
* **Audit Logging:** Granular, step-by-step database logging of every action and decision for compliance.
* **Workflow Versioning:** Snapshots of workflows that allow rolling back to previous state.

### 3. Backend Structure Required
* **Controllers:** `authController`, `workflowController`, `stepController`, `ruleController`, `executionController`, `logController`.
* **Services:** `ruleEngine.js`, `executionEngine.js`, `notificationService.js`.
* **Middleware:** JWT auth guards, RBAC role-checkers, and a unified global error handler parsing Mongoose errors to clean JSON.
* **Models:** `User`, `Workflow`, `Step`, `Rule`, `Execution`, `Log`.

### 4. Frontend Application Structure (`/frontend/pages/`)
1. **login.html & register.html:** Clean auth pages with demo credentials built-in.
2. **dashboard.html:** Professional analytics showing total requests, pending approvals, success rate, and Chart.js visual trends.
3. **workflows.html:** Datatable of leave policies with version control/rollback modals.
4. **builder.html:** The visual drag-and-drop graph canvas to string together Tasks/Approvals/Notifications.
5. **rules.html:** UI to configure routing conditions per step, featuring priority execution and AI generation.
6. **execution.html:** A submission form for an employee to paste their request payload and watch a live polling tracker follow the workflow progress.
7. **logs.html:** Filterable audit table showing exact historical footprints.
8. Include a shared `js/api.js` fetch wrapper that automatically handles JWT validation.

### 5. Implementation Instructions
1. Setup the project with `backend/` and `frontend/` folders.
2. Initialize the backend `package.json` with dependencies (`express`, `mongoose`, `bcryptjs`, `jsonwebtoken`, `cors`, `dotenv`, `mongodb-memory-server`).
3. Build the Backend models, controllers, and complex execution/rule engines.
4. Build the Frontend HTML pages and the professional CSS design system.
5. Create `run_demo.js` and `utils/seedData.js` to automatically boot the server and seed initial HR, Manager, User roles along with a sample workflow so the app is instantly testable out-of-the-box.
6. Progress logically using `task_boundary` and your artifact tools.
